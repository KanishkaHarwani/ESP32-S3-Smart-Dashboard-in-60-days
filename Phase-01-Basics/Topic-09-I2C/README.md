# Topic 09 — I²C

Your first real sensor. I²C is the bus you'll use for it — and for the BNO085 IMU too, if you explore that later in Phase 2/3.

## Why I²C for sensors

From Topic 08: I²C trades speed for simplicity — just 2 wires (plus power/ground), and multiple devices can share the same 2 wires because each has its own **address**. That's the right tradeoff for sensors, which send small amounts of data and don't need SPI's raw speed.

## The two wires

| Signal | Meaning |
|---|---|
| **SDA** | Serial Data — the actual data line, shared by every device on the bus |
| **SCL** | Serial Clock — keeps all devices in sync, generated by the master (your ESP32-S3) |

## Addresses: how one bus talks to many devices

Every I²C device has a fixed address (usually 7-bit, e.g. `0x38`) baked into the chip. When the ESP32-S3 wants to talk to a specific sensor, it sends that address first — every device on the bus "hears" it, but only the one with the matching address responds.

You'll often need to know a device's address before you can talk to it. Datasheets list it, but the fastest way to check on real hardware is to just ask the bus.

## Scanning the bus

A **bus scanner** loops through every possible address and asks "are you there?" — it's the standard first move whenever you wire up a new I²C device, and a great way to confirm wiring is correct before writing any real sensor code:

```cpp
Wire.begin();
for (uint8_t addr = 1; addr < 127; addr++) {
  Wire.beginTransmission(addr);
  if (Wire.endTransmission() == 0) {
    Serial.printf("Found device at 0x%02X\n", addr);
  }
}
```

If this finds nothing, it's almost always wiring (check SDA/SCL aren't swapped) or a missing pull-up resistor — not your code.

## Reading a real sensor

Most sensor libraries hide the raw I²C address/register details behind simple calls. This example uses an AHT20-style temp/humidity sensor (a common cheap I²C breakout — if yours is different, check its library docs, but the pattern is the same):

```cpp
#include <Adafruit_AHTX0.h>

Adafruit_AHTX0 aht;
aht.begin();

sensors_event_t humidity, temp;
aht.getEvent(&humidity, &temp);
Serial.printf("Temp: %.1fC  Humidity: %.1f%%\n", temp.temperature, humidity.relative_humidity);
```

> **Different sensor module?** If yours is a single-wire DHT11/DHT22 (3-pin, not 4-pin), that's *not* I²C — it uses its own timing-based protocol via the `DHT` library instead. Swap the include/library, the wiring and read pattern above still apply conceptually.
